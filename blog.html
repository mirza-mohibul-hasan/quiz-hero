Q-1: Differentiate among map, filter, forEach, and find.

Answer:

1. map(): In javaScript, .map() is a built-in method that allows us to create a new array by transforming each element of an existing array. The map() method applies a callback function to each element of the original array and returns a new array with the results.

2. filter(): In JavaScript, filter() is a built-in method that allows us to create a new array with elements which passed the condition or criteria. The filter() method applies a callback function to each element of the original array and returns a new array containing only the elements for which the callback function returns true.

3. forEach: In JavaScript, forEach() is a built-in method that allows you to iterate over the elements of an array and perform a specified action for each element. The forEach() method applies a callback function to each element of the array in order.

4. find(): In JavaScript, find() is a built-in method that allows you to find the first element in an array that satisfies a certain condition. The find() method applies a callback function to each element of the array and returns the first element for which the callback function returns true.

Q-2: Differentiate among Local Storage, Section Storage and Cookies.

Answer:

Local Storage, Session Storage, and Cookies are all client-side storage options in web browsers, but they differ in how long they store data, their storage capacity, and how they can be accessed.

1. Local Storage: Local Storage is a persistent storage mechanism that stores data without an expiration date, and the data remains stored even when the browser window is closed. Local Storage can store large amounts of data (up to 5-10 MB depending on the browser) and is accessible from any page in the same domain. Local Storage can be accessed using the localStorage object in JavaScript.

2. Session Storage: Session Storage is similar to Local Storage, but the stored data is cleared when the browser window is closed. Session Storage can store a moderate amount of data (up to 5-10 MB depending on the browser) and is only accessible from the page that created the data. Session Storage can be accessed using the "sessionStorage" object in JavaScript.

3. Cookies: Cookies are small text files that are stored on the client-side and sent to the server with every request. Cookies have an expiration date and can be set to expire after a specific time or when the browser window is closed. Cookies can store a limited amount of data (up to 4 KB) and are accessible from any page on the same domain that created the cookie. Cookies can be accessed using the "document.cookie" property in JavaScript.
